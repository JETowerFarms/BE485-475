Artifact Description

The artifact selected for this enhancement is the BE485-475 repository. It is a full-stack application for solar farm suitability analysis. It uses PostgreSQL with PostGIS and a Node/Express backend. The database stores farms, analysis results, raster data, incentives, and models. The artifact was created during prior academic work at Michigan State University.

This artifact was selected because it contains a database system with geospatial data and backend integration. It shows database design, query use, and data management.

View Repository

Justification and Enhancement

The database was improved in several areas. The schema was reviewed and duplication between schema and migration files was reduced. Query structure was improved by removing inefficient patterns and reducing unnecessary data retrieval. Validation before writes was strengthened.

The setup and migration process was improved to be consistent and repeatable. Storage and retrieval of farm data and cached analysis results were improved.

Enhancement Focus

  • Schema review and cleanup
  • Reduced duplication between schema and migration files
  • Improved query structure
  • Reduced unnecessary data retrieval
  • Stronger validation before writes
  • Consistent and repeatable setup and migration process
  • Improved storage and retrieval of farm data
  • Improved handling of cached analysis results
  • Removal of unsafe query patterns
  • Removal of hardcoded credentials

Course Outcome Alignment

This work demonstrates the ability to design and evaluate computing solutions using database practices. It shows analysis of an existing system and implementation of improvements. It uses PostgreSQL and PostGIS.

Security was improved by removing unsafe query patterns and hardcoded credentials. The planned outcomes were met, and no changes to the plan were required.

Reflection on the Enhancement Process

The process focused on improving an existing database system. Schema, migrations, and runtime logic were separated. Query design was improved. Spatial queries were made more efficient.

A challenge was keeping schema and queries consistent across the system. Changes required updates in multiple locations.

The final result is a database system that is more reliable, maintainable, and efficient.
