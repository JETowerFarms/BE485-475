Artifact Description

The artifact selected for this enhancement is the BE485-475 repository. It is a full-stack application for solar farm suitability analysis. It includes geospatial logic for polygon processing, grid generation, and spatial queries. The artifact was created during prior academic work at Michigan State University.

This artifact was selected because it contains multiple algorithms and data structures related to geospatial computation. It includes point-in-polygon checks, polygon area calculations, adjacency logic, and coordinate transformations.

Justification and Enhancement

The artifact was improved by organizing and consolidating these algorithms. Multiple implementations of point-in-polygon and area calculations were reduced into shared logic. A geometry utility layer was introduced to centralize these operations. Repeated computations were reduced by caching results such as polygon area and geometry structures. Data flow for polygon processing was improved to remove redundant steps.

Algorithm efficiency was improved. Expensive operations such as repeated object creation inside loops and duplicate point-in-polygon checks were reduced. Adjacency computation and hit-testing logic were identified as inefficient and improved through better structure and reduced repeated work. Coordinate transformation logic was made more consistent across the system.

Enhancement Focus

  • Geospatial computation
  • Shared geometry utilities
  • Point-in-polygon consolidation
  • Polygon area calculation reuse
  • Caching of repeated geometry results
  • Improved data flow for polygon processing
  • Reduced duplicate computations
  • Improved coordinate transformation consistency

Course Outcome Alignment

This work demonstrates the ability to design and evaluate computing solutions using algorithmic principles. It shows improvement of efficiency and structure in an existing system.

It also demonstrates the use of appropriate data structures for spatial data and graph relationships. The planned outcomes were met, and the artifact shows improved algorithm performance, organization, and maintainability.

Reflection on the Enhancement Process

The process focused on identifying repeated and inefficient algorithms. One result was consolidating multiple implementations into shared logic. This improved consistency and reduced duplication.

Another result was improving performance by reducing repeated computations. Caching and reuse were applied where possible.

A challenge was managing consistency across frontend and backend code. The same logic existed in multiple locations and required coordinated updates.

The final result is a system with more organized algorithms, improved efficiency, and clearer data flow.
