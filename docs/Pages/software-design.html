Artifact Description

The artifact selected for this enhancement is the BE485-475 repository. It comes from earlier academic work at Michigan State University and serves as a Michigan solar farm suitability and optimization tool. The project uses a React Native frontend with a Node and Express backend.

This artifact was selected because it shows real full-stack software development and includes a clear software design problem that could be improved through refactoring and stronger separation of concerns.

View Repository View Milestone Two Narrative

Justification and Enhancement

In Module One, FarmDescriptionScreen.js was identified as a very large file that combined too many responsibilities in one place. The enhancement focused on improving the structure and maintainability of the application by breaking the screen into smaller parts and moving repeated logic into shared modules.

The artifact showcases software development skills through the screen flow, shared utilities, backend routes and services, testing setup, and deployment support. The enhancement improved the artifact by moving the code toward a cleaner separation of concerns.

Enhancement Focus

  • Modular design
  • Refactoring
  • Component-based architecture
  • Maintainability
  • Testing readiness
  • Cleaner separation of concerns

Course Outcome Alignment

This enhancement supports the outcome related to designing and evaluating computing solutions because it improves the structure of an existing system instead of only adding features.

It also supports the outcome related to using well-founded techniques, skills, and tools because the project combines React Native, Node, Express, testing files, Docker support, and deployment infrastructure.

It also supports the security outcome in a partial way because the design now relies more on shared logic and backend-generated data.

Reflection on the Enhancement Process

This enhancement reinforced that software design problems are often caused by code that works but carries too many responsibilities in one place.

There were some features that were not functional until after this refactor. The exact source of the bug was not fully identified, but the work showed that clean code supports functional code.

The final result is a more maintainable structure that better supports future testing, extension, and reuse.
