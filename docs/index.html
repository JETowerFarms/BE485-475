Professional Self-Assessment

My CS 499 ePortfolio documents growth in technical and professional skills through the BE485-475 repository, a Michigan solar farm suitability and optimization tool built with a React Native frontend, a Node and Express backend, and PostgreSQL with PostGIS.

The portfolio focuses on full-stack software development, problem solving, database design, and applied analysis. It uses one artifact across the three required categories: software design and engineering, algorithms and data structures, and databases.

The software design enhancement improved structure and maintainability by breaking a large screen file into smaller parts and moving repeated logic into shared modules. The algorithms enhancement consolidated geospatial logic, reduced repeated computations, and improved consistency across the system. The database enhancement reduced duplication between schema and migration files, improved query structure and validation, and removed unsafe query patterns and hardcoded credentials.

Together, these materials present one artifact through software design, algorithms, database, and security work in a single GitHub Pages ePortfolio.

Artifact Overview

The selected artifact is the BE485-475 repository. It comes from prior academic work at Michigan State University and serves as a Michigan solar farm suitability and optimization tool.

This artifact was used across all three enhancement categories. The software design and engineering work focused on modularity and maintainability. The algorithms and data structures work focused on geospatial efficiency and shared logic. The database work focused on schema quality, validation, and reliable data access.
