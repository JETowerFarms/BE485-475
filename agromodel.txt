Updated Linear Optimization Model Outline (Michigan Agrivoltaics) – Full-Dev Only with Shared-Resource, Permitting, Interconnection, Agrivoltaic Geometry, Subsidy, Farmland-Preservation, and “Right Land” Constraints (Target 50% ITC)
1. Input handling
Land parameters: total available land ((A_{\text{tot}})); list of crop types ((j=1,\dots,J)) (each j represents a distinct crop); solar configuration choice ((\text{config}\in{\text{fixed},\text{tracker},\text{vertical},\dots})); land-use intensity ((\alpha_{\text{ac/MW}}(\text{config}))) = acres per MW_ac of solar. Typical values: fixed-tilt ≈5.5 acres/MW_ac, one-axis tracker ≈6.3 acres/MW_ac, two-axis ≈9.4 acres/MW_ac. (Assumption: \alpha_{\text{ac/MW}} reflects industry averages in Michigan; using these ensures capacity and land use are consistently linked.)
Project schedule: project life ((T)) in years; construction timing with CAPEX streams ((\text{CAPEX}_t)) for expenditures in year t (e.g. multi-year build); repower option flag ((\text{repower}\in{0,1})) indicating if a mid-life repowering (instead of full removal at T) is considered. (Assume T ~25-30 years for solar; repower option default 0 unless specified.)
Electricity price input (utility-scale revenue):
Retail reference (for context only): ((P^{\text{elec,ret}}_0 \approx 0.19\ $/\text{kWh})) – a typical retail electricity rate, used as a benchmark but not for selling project power. (Assumption: This highlights that selling power via net-metering might fetch ~$0.19/kWh, but our model uses a wholesale rate below this.)
Wholesale sell price: ((P^{\text{elec,sell}}0)) = initial power sale price (e.g. PPA price, $/kWh) for year 0; escalated by growth rate g{\text{elec}}: ((P^{\text{elec,sell}}_t = P^{\text{elec,sell}}0 (1+g{\text{elec}})^t)). (Assumption: P^{\text{elec,sell}}_0 is based on utility-scale solar PPA rates, significantly lower than retail – e.g. $0.05–$0.07/kWh.)
Curtailment/export factors: ((\phi^{\text{curt}}_t \in [0,1])) = fraction of potential generation not curtailed (1 = no curtailment); ((\phi^{\text{export}}_t \in [0,1])) = fraction of generation that can be exported (1 = no grid export limit). These reduce effective sales. (Assumption: Values are provided or scenario-based; e.g. \phi^{\text{curt}} = 0.95 if 5% curtailment expected, \phi^{\text{export}} = 1 unless export caps apply.)
Crop price inputs: ((P^{\text{crop}}{j,0})) = price for crop j at t=0 (in $/unit yield); crop-specific escalation ((g{\text{crop},j})) for prices. (Assumption: Based on USDA projections or inflation; e.g. 2% annual growth in crop prices.) For simplicity, yields and prices can be treated in aggregate revenue per acre terms if needed (but we keep them separate for fidelity).
Economic parameters: discount rate ((r)) for NPV (real terms, unless otherwise specified); inflation rate ((\pi)) if working with nominal values. If using nominal cash flows, convert a given nominal discount (or interest) to real: e.g., ((r_{\text{real}} = \frac{1+r_{\text{nom}}}{1+\pi} - 1)). Optionally, include tax rate ((\tau)) if tax effects are modeled (here mostly relevant for ITC usage, depreciation, etc., which we simplify by treating ITC as a direct credit). Debt financing terms if any: loan interest rate ((i_d)), loan tenor in years ((N_d)), and debt fraction (portion of upfront capital financed by debt). (Assumption: Unless analyzing financing explicitly, we might take r as WACC or a target return, and consider the project on an all-equity basis for simplicity.)
Land acquisition/lease costs:
Land purchase cost: If the project buys the land, upfront cost: ((\text{CAPEX}^{\text{land}}0 = A{\text{tot}} \cdot 6310)) $/acre. Here $6,310/acre is assumed as the land value. (Assumption: $6.3k/acre is an approximate Michigan farmland market price; this is a one-time cost at t=0 if land is purchased outright.)
Cropland rent baseline: ((\text{Lease}^{\text{crop}} \approx 152\ $/\text{acre-yr})) – the typical annual cash rent for farmland if used for crops (no solar). (Assumption: $152/acre-year based on regional average cropland rental rates; used as an opportunity cost benchmark for keeping land in farming.)
Solar land-lease parameters (FULL-DEV model – developer leases land from owner):
Lease rate (decision variable): ((L_{\text{solar}}\ [$/\text{acre-yr}])) = annual payment per acre to the landowner for land used by solar. This will be determined by the model.
Lease rate bounds: ((L_{\min} = 500,; L_{\max} = 1200\ $/\text{acre-yr})) to reflect plausible lease offers. (Assumption: Utility-scale solar leases in MI range roughly $500–$1200/acre-year; we constrain L_{\text{solar}} within this range.)
Lease payment stream: ((\text{LeasePay}t = A_s \cdot L{\text{solar}})) for each operational year t (t=1,...,T). This is the developer’s annual lease expense (and conversely the farmer’s lease income). If lease escalates with inflation or a fixed rate, we could include that (e.g., L_{\text{solar}}(t) = L_{\text{solar}} (1+g_{\text{lease}})^{t}), but for simplicity we assume a flat real lease rate.
Full-development gating: Lease payments occur only if solar is built. This is naturally enforced by the term $A_s$ in LeasePay (if $A_s = 0$, no lease paid). In other words, no scenario of “lease-only” without a solar project is considered. (This prevents the trivial solution of maximizing rent with no project: the model will require $A_s>0$ for any $L_{\text{solar}}>0$.)
Solar CAPEX parameters (per-acre via capacity and unit costs): All solar capital costs are converted to a per-acre basis using $\alpha_{\text{ac/MW}}$:
Installed PV cost (baseline): ((C^{\text{inst}}\ [$/W_{dc}] \approx 1.16)). This is the upfront cost per Watt_dc of solar (panels, inverters, installation). To relate to per-acre: first convert to $/MW_dc (1.16 $/W * $10^6$ W = $1.16M/MW_dc), then adjust for AC capacity per acre. For example, with a DC/AC ratio of ~1.3 and $\alpha_{\text{ac/MWac}} = 5.5$, 1 acre corresponds to about 0.18 MW_ac, which is 0.234 MW_dc; thus ~$0.234 \times 1.16M ≈ $271k/acre$. (Assumption: $1.16/W_dc is typical for utility-scale projects circa 2025; actual per-acre cost will depend on layout and DC/AC ratio. The model uses this to compute total cost.)
Site preparation: ((C^{\text{siteprep}}\ [$/\text{acre}] \approx 36{,}800)). Cost per acre for preliminary site work – grading (basic), fencing, access roads, drainage improvements, etc. (Assumption: $36.8k/acre derived from a typical site prep cost of $200k for ~5.5 acres (1 MW); covers fencing, road, trenching, etc.)
Grading (heavy earthwork): ((C^{\text{grade}}\ [$/\text{acre}] \in [5{,}524,; 11{,}048])). If the site is uneven, additional grading cost applies. We assume a range $5.5k–$11k per acre. (Assumption: Derived from $30k–$60k/MW for grading in rough terrain, spread over ~5.5 acres/MW. If land is flat, use low end; hilly, high end.)
Interconnection upgrade fraction: ((\alpha^{\text{inter}} \in [0.20,;0.40])). Fraction of base PV capital that must be spent on grid network upgrades (transformers, lines) to interconnect. E.g. $\alpha^{\text{inter}}=0.3$ means 30% additional cost on top of PV equipment for interconnect. (Assumption: Based on data that interconnection upgrades can add 20–40% to project CAPEX in weaker grid areas. If the project is small or grid is strong, this can be on the low end or zero.)
Solar OPEX / lifecycle parameters:
Base O&M: ((C^{\text{O&M}}\ [$/\text{kW_ac-yr}] \approx 11)). Annual operations & maintenance cost per kW_ac. Converting to per-acre: ((C^{\text{O&M}}\ [$/\text{acre-yr}])) = $11 \times 1000 \text{ kW/ac} / \alpha_{\text{ac/MW}}$. For example, with 5.5 acres/MW_ac, $C^{\text{O&M}} \approx 11 \times 1000 \times (1/5.5) \approx $2,000$/acre-yr. (Assumption: $11/kW-yr covers monitoring, routine maintenance, inverter servicing, and panel cleaning.)
Vegetation management: ((C^{\text{veg}}\ [$/\text{acre-yr}] \in [184,;265])). Cost to manage ground cover per acre per year (e.g. mowing, weed control or grazing management under panels). (Assumption: $184–$265 based on 2–3 mowings per year at $100/acre each plus some buffer; higher end if special pollinator habitat maintenance is needed.)
Insurance: ((\text{OPEX}^{\text{ins}}_t)) – annual insurance premiums (liability and casualty insurance for solar array). Could be a fixed $/acre or % of asset value. (Assumption: Typically $5–$10/kW-yr; at 5.5 acres/MW that’s ~$5,500–$11,000/MW or $1k–$2k/acre. We keep it as an input stream for flexibility.)
Inverter replacement: Inverter units typically need replacement once during project life. Let inverter lifespan ~12 years ((t_{\text{inv}} \in [10,15])). At year $t_{\text{inv}}$, incur ((\text{REPL}^{\text{inv}}{t{\text{inv}}} = A_s \cdot C^{\text{inv-repl}}\ [$/\text{acre}])). Assumption: C^{\text{inv-repl}} is ~10% of initial PV cost per acre. For instance, if initial per-acre PV cost was $270k, then inverter replacement ~$27k/acre at $t_{\text{inv}}$. We will include this one-time cost in year $t_{\text{inv}}$.
Decommissioning / remediation / salvage: (Costs at end-of-life, year T)
Decommissioning removal: ((C^{\text{decom}}\ [$/\text{kW}] \in [300,;440])) for dismantling and site clearance. Convert to $/acre: e.g. at $400/kW and using 5.5 acres/MW, ((C^{\text{decom}}\ [$/\text{acre}] ≈ 400 \times 1000 \times (1/5.5) ≈ $72,700)). (Assumption: $300–$440/kW based on industry estimates; covers dismantling racks, panels, inverter teardown, recycling logistics. We will use a mid value or scenario-based value.)
Decommissioning bond: ((C^{\text{bond}}\ [$/\text{acre}] \ge 10{,}000)). Upfront or phased financial assurance (escrow or bond) per acre to ensure funds for future decom. (Assumption: Michigan PA 116 requires a decommissioning bond; many localities set a minimum, e.g. $10k/acre, updated periodically. We treat $10k/acre as a minimum carrying cost or an included CAPEX if set aside at start.)
Site remediation: ((C^{\text{remed}}\ [$/\text{acre}] \approx 2{,}580)). Cost to restore soil and land after removal (decompacting soil, re-seeding with grass or crops). (Assumption: A moderate estimate; actual cost could include deep tillage, adding topsoil if eroded, etc.) Any additional costs (e.g., removal of access roads, fence) could be folded into this or decom. We assume this covers basic restoration.
Salvage value: ((C^{\text{salv}}\ [$/\text{acre}] \in [12{,}000,;13{,}000])). Materials recovered at end have value: steel racks, aluminum frames, copper wiring, possibly resellable panels. (Assumption: Rough estimate assuming ~0.2 $/W of salvageable value at end; for 1 MW_ac (≈5.5 acres), that’s ~$200k salvage, ~$36k/acre. We use a conservative one-third of that given recovery inefficiencies, ~$12k/acre salvage.)
Permitting / siting / interconnection inputs (expanded):
Zoning compliance flag: ((Z \in {0,1})) – indicates local zoning/ordinance requirements are satisfied. We require $Z=1$ for any solar development (if $Z=0$, the project as proposed is not permitted). (This is effectively a binary constraint representing whether the design complies with land-use and permitting rules such as special use permits, setbacks, etc.)
Zoning solar cap: ((A_{\text{zoning-allow}})) – maximum solar acreage allowed by local zoning. (Assumption: If a township ordinance caps utility-solar at e.g. 20% of farmland or an absolute acreage, that value is input here. If no explicit cap, $A_{\text{zoning-allow}} = A_{\text{tot}}$.)
Setback loss fraction: ((\delta_{\text{setback}} \in [0,1])) – fraction of the total land that must remain unused due to setbacks (distance from roads, property lines, structures). (Assumption: For a roughly rectangular site, setbacks might take ~5–15% of area; e.g. $\delta_{\text{setback}}=0.10$ means 10% of $A_{\text{tot}}$ is off-limits for panels, though it could still be used for crops or remain fallow.)
Height limit: ((h_{\max})) – maximum allowed height of solar structures (e.g. 20 ft by some local rule). Decision variable ((h_{\text{PV}})) = actual height used. (Assumption: For agrivoltaics, one might want taller racks for equipment clearance, but cannot exceed $h_{\max}$ without variance. $h_{\max}$ ~16–20 ft typical in ordinances. We will enforce $h_{\text{PV}} \le h_{\max}$.)
Easement/covenant exclusion: ((A_{\text{easement}})) – area of the site encumbered by easements or covenants (e.g. utility ROW, drainage ditches, conservation easements) where solar cannot be installed. (Assumption: This is measured or given; if none, $A_{\text{easement}}=0$. If e.g. a powerline crosses the property taking 2 acres, then $A_{\text{easement}}=2$.)
Wetland fill limits: ((W_{\text{fill}},;W_{\text{permit}})) – $W_{\text{fill}}$ = acres of wetlands the project design would fill or impact; $W_{\text{permit}}$ = threshold of wetland disturbance that triggers a special permit. (Assumption: Often if >~1/2 acre of wetland is disturbed, a permit is required. We will ensure $W_{\text{fill}}$ stays below that to avoid delays, if possible.)
Moratorium indicator: ((m \in {0,1})) – if 1, a local moratorium on new solar is in effect (meaning no project can proceed now). (Assumption: We treat $m$ as an input; $m=0$ normally. If $m=1$, the model will effectively force $A_s=0$ due to constraint.)
Interconnection capacity available: ((P_{\text{interconnect-available}})) – the maximum MW_ac that can be added at the grid connection without major system upgrades. (Assumption: Determined from utility interconnection studies; e.g. 5 MW available on the feeder before costly upgrades.)
Planned PV capacity: ((P_{\text{PV}})) – decision variable for solar farm capacity (MW_ac). This is linked to $A_s$ (see Section 2).
Earliest COD: ((\text{COD}{\min})) – earliest Commercial Operation Date (year) possible (e.g. due to interconnection queue). Decision variable $\text{COD}$ = actual COD year for the project. *(Assumption: If the interconnection process takes 3 years, and we are in 2025, $\text{COD}{\min}=2028$.)* We will enforce $\text{COD} \ge \text{COD}_{\min}$. (This affects cash flow start timing but is mainly a constraint to acknowledge.)
Interconnection upgrade cost & cap: $C_{\text{upgrade}}(P_{\text{PV}})$ = function for grid upgrade cost based on capacity. We will approximate this as $\alpha^{\text{inter}}$ fraction of solar CAPEX (linear). ((C_{\text{upgrade,max}})) = maximum tolerable upgrade cost. (Assumption: If $C_{\text{upgrade,max}}$ is, say, $1M, the project size will be limited such that required upgrades do not exceed that.) We will use a constraint to ensure upgrade costs stay under this cap.
Distribution penetration limits: Let $\text{PV_penetration}$ = fraction of local feeder peak load that this PV capacity represents. ((\text{PV_penetration}{\max})) = allowed maximum (e.g. 15% or 25%). We’ll ensure $\text{PV_penetration} \le \text{PV_penetration}{\max}$. (Assumption: This is to maintain voltage stability; if $P_{\text{peak}}$ is feeder peak load, $\text{PV_penetration} = P_{\text{PV}} / P_{\text{peak}}$.)
Inverter power factor: ((\text{pf}{\min})) – minimum power factor (inductive) required by utility (e.g. 0.95). Decision variable $\text{pf}$ = power factor at which inverters operate (typically 1.0 or slightly lagging to control voltage). We enforce $\text{pf} \ge \text{pf}{\min}$. (Assumption: Most utilities require at least 0.9–0.95; we assume 0.95 for model.)
Net-metering cap (if applicable): For behind-the-meter scenarios, ((P_{\text{NM}})) = portion of $P_{\text{PV}}$ under net-metering, and $\gamma_{\text{cap}}$ = fraction of the site’s peak load allowed. Constraint: ((P_{\text{NM}} \le \gamma_{\text{cap}} \cdot P_{\text{peak}})). (Assumption: Only relevant if modeling a smaller project offsetting on-site use. For utility-scale, $P_{\text{NM}}=0$.)
Drain tile damage metric: ((D_{\text{tile-damage}})) – an estimate of drainage tile lines impacted by construction (e.g. length of tiles crushed). ((D_{\max})) = maximum allowable tile disturbance (possibly 0 if we promise no damage, or some small length that can be repaired). We will constrain $D_{\text{tile-damage}} \le D_{\max}$. (Assumption: If the land has subsurface irrigation drainage, the project either avoids or repairs tiles. We treat $D_{\max}=0$ as ideal, meaning all tiles avoided or rerouted.)
Agrivoltaic design geometry: required clearances for farm equipment: panel minimum height ((\text{clearance})) and row spacing ((\text{row-spacing})). Given design variables for these, enforce ((\text{clearance} \ge \text{clearance}{\min})) and ((\text{row-spacing} \ge \text{row-spacing}{\min})). (Assumption: \text{clearance}{\min} e.g. 8 ft for tractors (or higher if combine use), \text{row-spacing}{\min} e.g. 20–30 ft to accommodate machinery and reduce shade on crops. These ensure the solar layout supports continued crop production adjacent or under arrays.)
Farmland-preservation / “remain farmland” inputs (PA 116 agricultural use program):
(Michigan’s PA 116 program provides tax credits for keeping land in ag use; solar can be allowed under certain conditions.)
Ag-use acreage variable: ((A_{\text{ag}})) – acres that remain in bona fide agricultural use. (E.g. acres planted in crops or enrolled in federal set-aside like CRP.)
Tillable, cleared acres: ((A_{\text{tillable,cleared}})) – acres of land that are cleared and could be tilled (excludes forest, wetlands, etc.). Used for determining income requirements on smaller parcels.
Gross annual ag income: ((\text{GrossAgIncome})) – gross income per year from agriculture on the land (crop sales, grazing, etc.). For parcels 5–40 acres, Michigan requires a minimum $200/acre of tillable land.
Definition of ag use: We only count land as $A_{\text{ag}}$ if it meets the criteria: production of plants or animals useful to humans, or participation in federal conservation programs (e.g. CRP). In model terms:
[
\text{AgUse} \in {\text{crop/livestock production, federal set-aside (CRP/etc.)}}.
]
(This is a note that $A_{\text{ag}}$ should correspond to such uses. In practice, $A_{\text{ag}}$ will equate to crop acres $A_c$ in our solution, since solar acres won’t count as ag use under PA 116.)
PA 116 solar compatibility flags: These binary inputs indicate compliance with specific program conditions that allow solar on PA 116 land:
((y_{\text{NRCS327}} \in {0,1})) – whether a pollinator habitat plan (NRCS practice #327) is implemented under the solar arrays. (Must be 1 (yes) to qualify.)
((y_{\text{return-to-ag-first-growing-season}} \in {0,1})) – whether there is a commitment to restore the land to agricultural production in the first growing season after the solar project ends. (Must be 1.)
((y_{\text{PA116-agreement-amended}} \in {0,1})) – whether the PA 116 agreement has been amended to allow this solar use. (Must be 1; an amendment is required before construction.)
((y_{\text{PA116-term-extended}} \in {0,1})) – whether the PA 116 agreement’s term has been extended by the years the land is used for solar. (Must be 1; typically the agreement is extended by the solar project duration.)
Pollinator habitat score: ((\text{Score}_{\text{poll}})) – score (0–100) from a pollinator habitat assessment for the solar site. To meet PA 116 condition, Score must be ≥76. (Assumption: 76 is the threshold defined by MI law for “pollinator-friendly solar” designation.)
Decommissioning financial assurance: ((B_{\text{decom}})) – the dollar amount of the decommissioning bond or escrow set aside. This must be ≥ the engineer’s cost estimate for removal: $\widehat{C}_{\text{decom}}$.
Tax credit deferment period: Let $\mathcal{T}{\text{defer}}$ be the set of years during which the land is used for solar and thus PA 116 tax credits are deferred (not paid). For each $t \in \mathcal{T}{\text{defer}}$, ((\text{PA116TaxCredit}_t = 0)). (Assumption: If the solar project operates years 1–20, then during those years the farmland tax credit is not received by the owner, per program rules. We include this to adjust the landowner’s cash flow, though it doesn’t affect the developer.)
“Right land” siting inputs (to ensure project is on appropriate land):
Land classification for solar: Partition $A_s$ into: farmland vs non-farmland, prime soils vs others, cleared vs forested. Use variables: ((A_{s,\text{farmland}})), ((A_{s,\text{nonfarmland}})), ((A_{s,\text{prime}})), ((A_{s,\text{cleared}})), ((A_{s,\text{forested}})). (Definitions: $A_{s,\text{farmland}}$ = solar acres on land currently zoned/used as farmland; $A_{s,\text{nonfarmland}}$ = solar acres on other land (brownfield, industrial, etc.); $A_{s,\text{prime}}$ = solar acres on prime agricultural soils (Class I/II); $A_{s,\text{cleared}}$ = solar acres on land that was already cleared/open (no forest); $A_{s,\text{forested}}$ = solar acres on forested land that would require tree clearing.)
Prime farmland cap: ((A^{\max}{s,\text{prime}})) = maximum allowable acreage of prime soil used for solar. *(Assumption: Policy or developer preference might limit solar on prime soils – e.g. at most X acres or X% of project. If we set $A^{\max}{s,\text{prime}} = 0$, it forbids using prime farmland.)*
Parcel set: If the project can choose among multiple parcels $p \in \mathcal{P}$, each parcel has attributes: slope, distance to grid, etc. We define those for completeness, though in many cases we’re focusing on a single site.
Slope of parcel $p$: $\text{slope}(p)$, with allowable max ((\text{slope}_{\max})) (e.g. 5% or 8% grade).
Distance from site to nearest substation or transmission: $\text{dist}(p,\text{substation})$, $\text{dist}(p,\text{transmission})$; define ((D_{\max})) as max acceptable distance (e.g. 1 mile).
Hosting capacity of parcel: $H(p)$ (MW) – how much PV that location can accommodate on the grid side.
(Assumption: We generally will pick one parcel or aggregate parcels such that these conditions are met, rather than actually selecting among many in the optimization.)
Farmland-only siting assumption: We assume the project is on farmland (since this is an agrivoltaics scenario). So we intend to use only farmland for solar: no non-farmland. Thus $A_{s,\text{nonfarmland}}$ will be forced 0, and $A_s = A_{s,\text{farmland}}$. (This is a modeling choice aligning with analyzing impacts on farmland; if the project could use a non-farm brownfield, that’d be a different scenario.)
No forest clearing: We assume no clearing of mature forests for solar. Thus $A_{s,\text{forested}} = 0$, and $A_s = A_{s,\text{cleared}}$. (All solar goes on land already cleared or in crops/pasture.)
Slope constraint: Require each chosen solar parcel $p$ to have $\text{slope}(p) \le \text{slope}_{\max}$ (e.g. no panels on slopes >5%). This prevents trying to utilize steep land that would incur high cost or erosion issues. (If the available land has portions > slope_max, those portions effectively can’t be used in $A_s$.)
Distance constraint: Require $\text{dist}(\text{site},\text{substation}) \le D_{\max}$ (and/or similar for transmission). This ensures the site is close enough to grid access to be viable. (Assumption: $D_{\max}$ might be ~1–2 miles; beyond that, line extension costs might be prohibitive unless accounted for in $\alpha^{\text{inter}}$.)
State setback rules: Enforce explicit minimum setbacks (if not already captured by $\delta_{\text{setback}}$): e.g.
((\text{setback}_{\text{road}} \ge 50\ \text{ft})) from road ROW,
((\text{setback}_{\text{nonparticipant line}} \ge 50\ \text{ft})) from adjacent property lines,
((\text{dist}(\text{fence},\text{occupied structure}) \ge 300\ \text{ft})) from any occupied home.
These are typical values from guidelines to minimize nuisance. (We include them as constraints to ensure design feasibility. In practice, these reduce usable area – already reflected in $\delta_{\text{setback}}$, but listing them ensures compliance checks.)
Parcel hosting capacity: If multiple parcels $p$, ensure the allocated PV on each does not exceed that parcel’s hosting capacity: $(P_{\text{PV}}(p) \le H(p)\ \forall p)$. In a single-site model, this is equivalent to the grid capacity constraint we already have for the whole site.
Subsidy qualification inputs (target 50% ITC):
The Inflation Reduction Act (IRA) provides a base 30% Investment Tax Credit for solar, plus potential adders. We target achieving a 50% ITC via adders.
((\eta_{\text{ITC}} \in [0,1])) – effective ITC fraction of eligible solar CAPEX. We aim for $\eta_{\text{ITC}} = 0.50$ (50%).
ITC adder qualification binaries:
((y_{\text{PWA}} \in {0,1})) – Prevailing Wage & Apprenticeship requirements met. (Must be 1 to avoid ITC reduction; assume we will comply, so $y_{\text{PWA}}=1$.)
((y_{\text{DC}} \in {0,1})) – Domestic Content requirement met (for +10% ITC). (We set $y_{\text{DC}}=1$ to get the adder, assuming using enough US-made components.)
((y_{\text{EC}} \in {0,1})) – Energy Community location met (for +10% ITC). (We assume the site qualifies or we select a site that does, so $y_{\text{EC}}=1$.)
With all three = 1, the ITC could be 30% base + 10% + 10% = 50%. (Assumption: Possibly one of the adders is not used if base 30% requires PWA anyway, but essentially we ensure 50% total. If only two adders apply, base 30 +210 = 50. We fix $\eta_{\text{ITC}}=0.50$ in our base model to simulate achieving this target.)*
2. Data preprocessing
Unit consistency: Convert all inputs to a common basis (annual real dollars, acres, MW_ac, etc.):
All cash flows in constant dollars (use real discount rate $r$). If given a nominal discount and inflation, adjust $r$. For example, if $r_{\text{nom}}=8%$ and $\pi=3%$, then $r = (1.08)/(1.03) - 1 \approx 4.85%$.
Ensure capacity in MW_ac and energy in kWh. (If energy yield given per kW_dc, convert using DC/AC ratio.)
Land in acres throughout. (If some data was per hectare, convert: 1 ha = 2.471 ac.)
Cost units: convert $/W to $/acre as needed using $\alpha_{\text{ac/MW}}$. Convert $/kW to $/acre similarly.
This step ensures that when we form linear coefficients, everything aligns (e.g., we don’t mix up W_dc vs W_ac or nominal vs real dollars).
Time index conversion: If any rates are monthly or quarterly, convert to annual. (Notably, if a monthly discount rate $r_m$ is provided, use $r = (1+r_m)^{12} - 1$ to get annual.) Our model primarily works on an annual timestep.

3. Decision variable definitions
Continuous decision variables:
((A_s \ge 0)) – acres allocated to solar (full-development). This is a primary decision: how much land to use for PV.
((A_{c,j} \ge 0)) – acres allocated to crop $j$ (for each $j=1,\dots,J$). The total crop acres $A_c = \sum_j A_{c,j}$. (We could also have one aggregated crop variable if treating all crops the same, but keeping by type allows crop rotations or different profits.)
(If needed: ((A_c \ge 0)) – total crop acres, but we can use $A_c$ as a shorthand = $\sum_j A_{c,j}$.)*
((P_{\text{PV}} \ge 0)) – solar PV capacity (MW_ac) installed. This is tied to $A_s$ by $P_{\text{PV}} = A_s/\alpha_{\text{ac/MW}}$. Including it as a variable makes it easier to impose MW-based constraints (like interconnection limits).
((h_{\text{PV}} \ge 0)) – height of PV panels (feet or meters). Used to enforce $h_{\text{PV}} \le h_{\max}$. It might also implicitly affect $f_j(\cdot)$ for crop yield (taller = less shading on crops), but we did not explicitly model that dependency except to ensure a minimum for equipment.
((\text{pf} \in [0,1])) – inverter power factor. We expect the solution to simply set $\text{pf} = \text{pf}_{\min}$ if that is required (no objective incentive to go higher except maybe to avoid curtailment which we haven’t modeled, so it will likely just meet minimum).
((\text{COD} \ge 0)) – Commercial Operation Date (year). Could be a continuous variable in our linear model, but effectively we will set a lower bound $\text{COD}{\min}$. Since this doesn’t directly affect objective (unless we discounted cash flows differently), we include it mainly to ensure $\text{COD} \ge \text{COD}{\min}$. We won’t need to optimize over COD; we can set COD = COD_min if project built as soon as allowed. (In a more detailed model, one might choose to delay COD if it aligns better with prices or budgets, but we assume as soon as ready.)
Farmland preservation continuous variables:
((A_{\text{ag}} \ge 0)) – acres considered in agricultural use. Likely we will have $A_{\text{ag}} = A_c$ (since only crop acres count). If we had some solar acres still counting (like grazing under panels), that could add, but for now, no – unless specified otherwise, $A_{\text{ag}} = A_c$.
((A_{\text{tillable,cleared}} \ge 0)) – acres of land that are both cleared and could be tilled. For our site, if it’s all farmland, $A_{\text{tillable,cleared}}$ might equal $A_{\text{tot}}$ minus any truly non-tillable areas (like wetlands, but those we excluded or they count as cleared? Actually wetlands are not “tillable”). For simplicity, if the site was all farmland, $A_{\text{tillable,cleared}} = A_{\text{tot}}$ minus forest/wetland area. We can treat it as a known input.
((\text{GrossAgIncome} \ge 0)) – gross annual agricultural income (in $). In practice, $\text{GrossAgIncome} = \sum_j A_{c,j} Y_{j,t} P^{\text{crop}}_{j,t}$ (for a given year’s yields and prices). If yields/prices vary annually, this is time-dependent; however, the PA 116 rule uses a fixed threshold $200/acre which likely is evaluated yearly. We can use initial year or an average year for this constraint. We’ll implement it as if it must hold initially (or in a representative year).
Land-class allocation variables: (to track siting categories for constraints)
((A_{s,\text{farmland}} \ge 0)) – solar acres on farmland.
((A_{s,\text{nonfarmland}} \ge 0)) – solar acres on non-farmland. (We will enforce $A_{s,\text{nonfarmland}}=0$ for our scenario.)
((A_{s,\text{prime}} \ge 0)) – solar acres on prime soil.
((A_{s,\text{cleared}} \ge 0)) – solar acres on cleared land.
((A_{s,\text{forested}} \ge 0)) – solar acres on forested land. (We will enforce $A_{s,\text{forested}}=0$.)
These are not independent decisions but are linked to $A_s$. We include them to impose constraints like $A_{s,\text{prime}} \le A^{\max}{s,\text{prime}}$, etc. Relationships: $A_s = A{s,\text{farmland}} + A_{s,\text{nonfarmland}} = A_{s,\text{cleared}} + A_{s,\text{forested}}$, etc., will be enforced.
Lease rate variable:
((L_{\text{solar}} \ge 0)) – solar lease payment per acre ($/yr). This is unusual as a decision variable in a single-level model (price usually outcome of a negotiation between two parties). But here, we include it to determine the maximum feasible lease the developer can pay while meeting their financial criteria. It will be the objective of Stage A optimization. We have bounds $L_{\min}, L_{\max}$ as given.
Binary (0–1) decision variables:
((y_k \in {0,1})) – configuration choice binaries, if applicable. One for each config type k (fixed, tracker, etc.), with $\sum_k y_k = 1$. If $y_k=1$, that config is selected and its $\alpha_k$ applies. (If we assume a fixed config upfront, we can omit these to keep the model linear.)
((z_{\text{repower}} \in {0,1})) – repowering decision at end of project. $z_{\text{repower}}=1$ might indicate we choose to replace panels and continue (instead of decommission at year T). If $z_{\text{repower}}=0$, we fully decommission at T. (For simplicity, we may not use this in the optimization, as we have not built out the repower scenario cash flows. It’s noted for completeness if exploring extend vs terminate.)
Permitting compliance binaries:
((y_{\text{glare}} \in {0,1})) – whether glare mitigation requirements are satisfied (e.g., glare study done, mitigations in place). Many ordinances require “no glare” for drivers/neighboring properties. We assume we will comply, so we will set $y_{\text{glare}}=1$ in any feasible solution.
((y_{\text{decom-plan}} \in {0,1})) – whether a decommissioning plan has been submitted/approved. Typically required by counties before construction. We also set $y_{\text{decom-plan}}=1$ (the model must include a decom plan). These binaries effectively must be 1 for $Z=1$. We will enforce them.
Subsidy qualification binaries: ((y_{\text{PWA}}, y_{\text{DC}}, y_{\text{EC}} \in {0,1})) as defined earlier. We will enforce each = 1 to achieve the 50% ITC. (We could allow the model to choose not to meet them if, say, domestic content raises cost, but we have not modeled a cost penalty for domestic content. So we simply assume they are met.)
PA 116 compliance binaries: ((y_{\text{NRCS327}}, y_{\text{return-to-ag-first-growing-season}}, y_{\text{PA116-agreement-amended}}, y_{\text{PA116-term-extended}} \in {0,1})). All of these must equal 1 to satisfy PA 116 solar use conditions. We will treat them as requirements (the model won’t choose them; we set them =1 as part of feasible solution definition). If any were 0, the project would violate the PA 116 agreement unless the land was withdrawn (which is another scenario not covered here).
Zoning pathway binary: ((Z \in {0,1})) – as above, indicates if the project meets zoning. We require $Z=1$. We won’t let the model choose $Z$; rather, $Z$ is an input reflecting whether the design is permissible. We include it as a binary in case we want to enforce $Z=1$ through meeting other constraints (like $y_{\text{glare}}=1$, etc., might be part of what makes $Z=1$ feasible).

Payback period: number of years until cumulative cash flow turns positive. Again, not a decision factor, but we can calculate it from the NCF stream afterward for insight (or enforce a constraint like payback <= N years by linear approximation, but we won’t here).

$\text{NCF}_t^{\text{(dev)}}$ is the developer’s net cash flow in year $t$ (solar revenues minus solar costs, including lease payments).
$\text{NPV}{\min}^{\text{(dev)}}$ is a threshold set by the developer. Commonly, this might be 0 (break-even NPV at discount rate $r$), representing that the developer at least doesn’t lose money (and effectively achieves IRR = $r$). We can use 0 for a break-even requirement. If the developer requires a positive NPV (for profit) or a higher IRR, we could set $\text{NPV}{\min}$ to some positive value, but that would tighten the constraint.
This constraint is linear because $\text{NCF}t^{\text{(dev)}}$ is linear in the decision variables ($A_s$ and $L{\text{solar}}$ primarily). We can explicitly write $\text{NCF}_t^{\text{(dev)}} = \text{Rev}^{\text{solar}}_t - (\text{OPEX}^{\text{solar}}_t + \text{LeasePay}_t + \text{REPL}_t + \text{DECOM}_t - \text{SALV}t)$ for each $t$. When we plug in $\text{LeasePay}t = A_s L{\text{solar}}$, etc., the left side becomes a linear expression in $A_s$ and $L{\text{solar}}$ (and possibly other variables like if $A_s$ affects revenue linearly, which it does, etc.). So it is suitable as a linear constraint.
Intuitively, this constraint ensures that if $L_{\text{solar}}$ is set very high, the developer’s NPV (which decreases as lease payments increase) does not fall below the acceptable threshold. The model will increase $L_{\text{solar}}$ until this NPV constraint is just binding (or another constraint, like max lease or land available, stops it). Thus we find the maximum rent that still leaves the developer with $\text{NPV}_{\min}$.
(We will set $\text{NPV}_{\min}^{\text{(dev)}} = 0$ for break-even unless instructed otherwise. This essentially means the developer is willing to undertake the project if it at least breaks even at discount rate $r$. In reality, a developer might want NPV > 0 or require a certain IRR, but for maximizing lease, we’ll push them to their break-even point.) Other constraints from Section 7 (land availability, interconnect, 51% rule, etc.) also apply in Stage A, because they can restrict $A_s$ and other decisions, which in turn affect feasibility and NPV. Important: We should avoid the trivial solution $A_s = 0$ with a high $L_{\text{solar}}$. If $A_s=0$, developer NPV is 0 (no cost, no revenue), satisfying NPV ≥ 0, and the objective could drive $L_{\text{solar}}$ to $L_{\max}$ with no project built – which is not a valid scenario (no project actually occurs). To prevent this, we implicitly assume we are considering solutions with $A_s > 0$. In practice, we can add a tiny constraint like $A_s \ge \epsilon$ or simply note that a solution with $A_s=0$ and $L_{\text{solar}}$ high is not implementing a project. We will exclude that by focusing on solutions where some solar is built. (One way is to require $A_s \ge 1$ acre if any solar at all – since partial acreage below that is unrealistic anyway. We’ll assume $A_s$ will naturally go >0 because otherwise the farmer would just farm all land and $L_{\text{solar}}$ is irrelevant.) Stage A outcome: The solution will give:
$L_{\text{solar}}^* =$ maximum feasible lease rate,
and associated $A_s^$ (how many acres of solar at that lease).
Often, if solar is highly profitable, $L_{\text{solar}}^$ might hit the upper bound $L_{\max}$. If solar is marginal, $L_{\text{solar}}^*$ might be much lower or even hit $L_{\min}$ if NPV constraint isn’t the active one (but typically, if we’re maximizing $L$, the NPV or some other limit will bind, not the min).
It’s possible that the farmland preservation constraints (like 51% ag use) force a limit on $A_s$ which in turn limits how high $L$ can go (because if $A_s$ is smaller, developer’s NPV might be actually smaller scale – although interestingly, a smaller project could have higher $L$ since less capacity might just meet NPV=0 at a higher rent per acre). The interplay is complex, but the solver will find the optimum combination of $A_s$ and $L_{\text{solar}}$ within all constraints.
6.2 Land-allocation competition objective (Stage B)
In Stage B, we fix $L_{\text{solar}} = L_{\text{solar}}^*$ (the optimal lease from Stage A), and re-optimize to allocate land between solar and crops to maximize the landowner’s benefit (or overall system benefit). Objective (Stage B): We choose an objective representing the “competition” for land between uses. A reasonable choice is to maximize the farmer’s Net Present Value (or net benefit), since the developer’s profit at this point is essentially zero by design (they’re at break-even paying $L_{\text{solar}}^*$). Maximizing farmer NPV will effectively maximize total welfare in this setup, because any additional solar beyond the developer’s break-even benefits the farmer without hurting the developer (it would actually give the developer positive NPV if we didn’t constrain them, but we already fixed $L$ at break-even level; if we allow more $A_s$, the developer’s NPV would go >0 which is fine as long as the constraint is ≥0, not =0). For simplicity, we can think of it as: each acre can either be used for crops or for solar.
If used for solar, the farmer gets $L_{\text{solar}}^*$ per year (and loses whatever crop profit that acre would have provided).
If used for crops, the farmer gets the crop profit (and no lease).
We want to allocate acres to whichever gives higher present value, up to the limits required by policy (like must keep 51% in ag, etc.). So our Stage B objective could be linear:

$A_s^*$ (possibly different from Stage A’s $A_s$, which was the size that made developer break-even at max $L$ – developer could still be okay doing a bit more area since any extra gives them positive NPV at same lease, but they wouldn’t object if allowed).

All constraints from Section 7 still apply in Stage B.
The developer NPV constraint is no longer needed because we already fixed $L$ from that solution (and any increase in $A_s$ now would only increase developer NPV, so they remain ≥0). We effectively satisfy it by construction, so we can drop it in Stage B.
But we must keep the farmland-preservation constraints, land limit, etc., as those guide the allocation.
In particular, the 51% ag use constraint will likely bind: it says at most 49% of land can go to solar. So we expect $A_s$ to hit that limit (if solar lease is more profitable than crops). If crop profit per acre was very high (maybe specialty crops, etc.), the model might not even use all allowed solar acreage, but that’s unlikely for standard commodity crops.
We can also consider alternative objectives:
We could maximize total combined NPV of farmer+developer. But since developer NPV is pegged at ~0 (at break-even), maximizing combined NPV is essentially maximizing farmer NPV plus a constant (the developer stays at 0, or if $A_s$ increased, developer NPV would go positive, contributing to combined – but if we allow that, the combined optimum might use 100% for solar because now even developer gains; however, the developer NPV constraint is ≥0 not exactly 0, so yes, combined NPV would increase if we enlarge $A_s$ beyond 51% constraint until some other limit. But 51% will stop it anyway).
So maximizing farmer NPV or combined NPV yields the same land allocation since the binding constraint will be farmland requirement, not the objective trade-off. If there were no 51% rule, combined NPV would just maximize $A_s$ until maybe land or budget runs out, giving developer >0 NPV which is fine because we only constrained ≥0. But farmland rule or others might then be the only thing preventing all land to solar. We’ll see that in sensitivity possibly.
So, Stage B outcome:
Optimal $A_s^*$ (which likely will be $0.49 A_{\text{tot}}$ if 51% rule active and solar lease is lucrative, or possibly $A_{\text{tot}}$ if the rule didn’t exist or is less stringent under some scenario).
Breakdown of crop acres $A_{c,j}^*$. Possibly, if one crop is more profitable than another, the model might allocate all $A_c$ to the most profitable crop ($\Pi_{\text{crop},j}$ highest). If there’s a rotation or requirement to grow multiple crops, we haven’t enforced that except through possibly the gross income rule (which just says total income must exceed threshold, which likely it will if any decent cropping is done on 51% land). We could include optional constraints like at least some of each crop if diversification is needed, but not required by the prompt. So the model might choose the best crop and allocate all crop land to it. We can restrict if needed (e.g., maybe corn and soy rotation – but for optimization we might just treat them generically if they have similar economics).

Integrality: If MILP (config choices $y_k$ present), those $y_k$ are binary. We can solve with a MIP solver. If we fixed config, it’s a pure LP. Other binaries like the compliance ones are forced =1 anyway (which essentially removes their binary nature as a decision – they’re fixed parameters in the solve). So the only binary that might affect solve is $y_k$. If we stick to one config, no binary at all and it’s just an LP.
Feasibility check: If no solution exists with $A_s>0$ satisfying NPV ≥ 0, then it means even at $L_{\text{solar}}=L_{\min}$ developer can’t break even (i.e. project not viable under given prices/costs unless maybe farmer accepted lower lease). Unboundedness check: If developer NPV constraint is easily satisfied even at $L_{\max}$, the objective might try to increase $L_{\text{solar}}$ to the upper bound $1200. If $\beta=1$ (i.e. solar acres are also all ag via grazing), then constraint becomes $A_c + A_s \ge 0.51 A_{\text{tot}}$. Combined with land constraint $A_c + A_s \le A_{\text{tot}}$, the range of feasible changes. If $A_s$ had been limited to 49% because $\beta=0$, now $\beta=1$ might allow $A_s$ to go up to 100% as long as $A_c$ is 0 because then $A_s = 100%$ gives LHS $0 + 1*100% = 100% \ge 51%$ satisfied. So in that extreme, they could do entire farm solar and claim because they graze sheep, it's still agriculture, thus fulfilling program.
So yes, if agrivoltaics got official recognition as ag use, farmland-preservation program might not restrict solar much at all. Then optimal solution would be use all land for solar (since presumably the animal/grazing income is small relative to solar but enough to count as use).
That scenario basically nullifies 51% rule. We can mention that as a potential regulatory change and its impact (massive increase in solar potential on farmland).
Perturb other constraints:
If we had ROI or DSCR constraints (which we didn’t enforce), but say if developer insisted on, e.g., IRR 8% rather than just break-even 5%, that would be like requiring NPV ≥ some positive number (makes it harder to pay high rent). If we linearized that as, e.g., NPV ≥ X% of CAPEX (like requiring positive NPV ratio), we could simulate: basically developer would pay less rent to achieve their target profit. Then $L_{\text{solar}}$ optimum would be lower. The farmer gets less, developer keeps some profit. We can discuss that qualitatively: in real negotiations, developer might not actually give all profit to landowner (they aim for some IRR). Our model giving them break-even is a theoretical max landowner could demand if developer just at threshold. If developer needed, say, $N profit, that could be set and then $L$ would come out lower. So farmland owner might not actually get $L_{\max}$ from Stage A if developer had pricing power or alternatives. But if farmland owners have strong bargaining and many developers bidding, perhaps they can push it to near break-even levels.
If DSCR minimum was added, that might limit how much debt they can raise if rent is high (because high rent = low cash flow cushion). If DSCR<1, project couldn’t get financed. If our solution had borderline DSCR (maybe if we gave them any profit, likely DSCR fine because NPV=0 at 5% discount implies around 1.0 average DSCR maybe; typically lenders want >1.2, so developer might actually want some cushion -> again needing lower rent).
Could mention: “If financing constraints require, e.g., 20% project NPV margin, the lease would need to be set lower such that developer retains some profit. This would shift some value from landowner back to developer to satisfy lenders. The model could incorporate that by setting $\text{NPV}{\min}$ to a positive value or adding a DSCR constraint. The effect would be to reduce $L{\text{solar}}$ and possibly reduce solar acreage if viability is borderline. Essentially, landowner’s maximum rent is somewhat theoretical – in practice developer will ensure a safety margin, meaning actual negotiated lease might be lower than the absolute max found here.”
We’d provide scenario comparison perhaps in a table:
e.g.
Base case: $L^*=$800, A_s=49 ac (4.9 MW), farmer NPV $X, developer NPV $0 (IRR 5%)
High price scenario: $L^*=$1200 (cap), A_s=49 ac, farmer NPV higher, developer still break-even (or slight positive because cap prevented going higher).
Low price scenario: no solar (not viable at minimum rent).
etc.
Focus on interesting ones: farmland rule off, prime farmland avoid, etc.
